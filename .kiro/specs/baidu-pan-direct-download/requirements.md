# 需求文档

## 简介

百度网盘直链获取工具是一个独立的网页工具，用户通过输入百度网盘分享链接和提取码，工具调用百度网盘 API 获取真实下载直链，用户可以复制链接后使用 IDM 等下载工具进行高速下载。整个工具为纯前端实现，无需服务器中转。

## 词汇表

- **工具（Tool）**：独立的 HTML 网页应用，包含用户界面和 API 调用逻辑。
- **API 调用器（ApiCaller）**：负责构造并发送百度网盘 API 请求、解析响应的模块。
- **分享链接（Share Link）**：百度网盘的文件分享链接，格式为 `https://pan.baidu.com/s/xxxxx`。
- **提取码（pwd）**：访问百度网盘分享链接所需的访问密码，通常为 4 位字符。
- **真实下载链接（Direct Link）**：由百度网盘 API 返回的、可直接用于下载文件的 URL。

## 需求

### 需求 1：用户输入界面

**用户故事：** 作为用户，我希望有一个简洁的输入界面，以便输入分享链接和提取码。

#### 验收标准

1. THE 工具 SHALL 提供一个文本输入框用于输入百度网盘分享链接。
2. THE 工具 SHALL 提供一个文本输入框用于输入提取码。
3. THE 工具 SHALL 提供一个"获取下载链接"按钮用于触发 API 调用。
4. WHEN 用户未输入分享链接，THE 工具 SHALL 禁用"获取下载链接"按钮。

---

### 需求 2：分享链接解析

**用户故事：** 作为用户，我希望工具能自动解析我输入的分享链接，以便提取必要的参数。

#### 验收标准

1. WHEN 用户输入完整的百度网盘分享链接（如 `https://pan.baidu.com/s/1xxxxx`），THE 工具 SHALL 从 URL 中提取 `surl` 参数。
2. WHEN 用户输入短链接（如 `1xxxxx`），THE 工具 SHALL 将其识别为 `surl` 参数。
3. IF 输入的链接格式不正确，THEN THE 工具 SHALL 显示错误提示"分享链接格式不正确"。

---

### 需求 3：调用百度网盘 API 获取下载链接

**用户故事：** 作为用户，我希望工具能调用百度网盘 API 获取真实下载链接，以便我可以使用 IDM 下载。

#### 验收标准

1. WHEN 用户点击"获取下载链接"按钮，THE API 调用器 SHALL 使用分享链接和提取码向百度网盘 API 发送请求。
2. THE API 调用器 SHALL 在请求头中携带必要的 `Referer` 和 `User-Agent` 信息。
3. WHEN API 返回成功响应（errno 为 0），THE API 调用器 SHALL 解析并返回真实下载链接。
4. IF API 返回错误响应（errno 不为 0），THEN THE API 调用器 SHALL 显示对应的错误信息。
5. IF 网络请求超时（超过 15 秒），THEN THE API 调用器 SHALL 显示"请求超时，请重试"提示。

---

### 需求 4：显示下载链接

**用户故事：** 作为用户，我希望工具能清晰地显示获取到的下载链接，以便我可以复制使用。

#### 验收标准

1. WHEN 下载链接获取成功，THE 工具 SHALL 在页面上显示完整的下载链接。
2. THE 工具 SHALL 提供一个"复制链接"按钮，方便用户一键复制。
3. WHEN 用户点击"复制链接"按钮，THE 工具 SHALL 将下载链接复制到剪贴板。
4. WHEN 复制成功，THE 工具 SHALL 显示"复制成功"提示。

---

### 需求 5：错误处理

**用户故事：** 作为用户，我希望工具能清晰地提示错误信息，以便我知道如何解决问题。

#### 验收标准

1. IF 提取码错误，THEN THE 工具 SHALL 显示"提取码错误，请检查后重试"。
2. IF 分享链接已失效，THEN THE 工具 SHALL 显示"分享链接已失效"。
3. IF 分享链接不存在，THEN THE 工具 SHALL 显示"分享链接不存在"。
4. IF 文件已被删除，THEN THE 工具 SHALL 显示"文件已被删除"。
5. IF 发生未知错误，THEN THE 工具 SHALL 显示"未知错误（错误码：{errno}）"。

---

### 需求 6：用户界面体验

**用户故事：** 作为用户，我希望工具界面简洁美观，操作流畅。

#### 验收标准

1. THE 工具 SHALL 采用响应式设计，在不同屏幕尺寸下都能正常显示。
2. WHEN 正在获取下载链接时，THE 工具 SHALL 显示加载动画，并禁用"获取下载链接"按钮。
3. WHEN 操作完成或发生错误，THE 工具 SHALL 恢复按钮可点击状态。
4. THE 工具 SHALL 使用清晰的视觉层次，突出重要信息。

---

### 需求 7：安全性

**用户故事：** 作为用户，我希望工具在本地运行，不泄露我的信息。

#### 验收标准

1. THE 工具 SHALL 仅在用户主动点击"获取下载链接"按钮后才发起 API 请求。
2. THE 工具 SHALL 不将用户输入的任何信息发送至第三方服务器。
3. THE 工具 SHALL 所有网络请求的目标域名仅限于 `pan.baidu.com` 及其子域名。
4. THE 工具 SHALL 不加载任何外部 CDN 资源或远程脚本。
